# ls-web-open: 웹 대시보드 서버 실행

터틀 트레이딩 봇 웹 대시보드를 실행합니다. (`python main.py`)

## 실행 절차

### 1단계: .env 파일 확인

프로젝트 루트에 `.env` 파일이 있는지 확인합니다.

- **없으면**: `.env.example`을 복사하여 `.env`를 생성하고, 사용자에게 API 키를 입력받습니다.
  - 필수 키: `LS_APPKEY`, `LS_APPSECRETKEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`
  - 각 키를 하나씩 질문하되, 빈 값(엔터)이면 placeholder를 유지합니다.
- **있으면**: 다음 단계로 진행합니다.

### 2단계: 포트 충돌 확인

8000번 포트가 이미 사용 중인지 확인합니다:

```bash
lsof -ti:8000
```

- 이미 사용 중이면 사용자에게 알립니다: "8000번 포트가 이미 사용 중입니다. 기존 프로세스를 종료하고 다시 시작할까요?"
- 사용자가 동의하면 기존 프로세스를 종료(`kill`)한 뒤 3단계로 진행합니다.

### 3단계: 서버 실행

프로젝트 루트에서 백그라운드로 서버를 실행합니다:

```bash
python main.py
```

- `run_in_background: true`로 실행합니다.
- 3초 대기 후 출력 로그를 확인하여 정상 시작 여부를 판단합니다.

### 4단계: 결과 보고

서버 시작 로그를 확인하고 결과를 보고합니다:

**성공 시:**
```
서버가 실행되었습니다: http://localhost:8000
- 해외주식 봇: {상태}
- 해외선물 봇: {상태}
- 텔레그램 봇: {상태}
```

**실패 시:**
- 에러 로그를 보여주고 원인을 분석합니다.
