# ls-commit: 작업 내역 상세 커밋

현재까지의 작업 내용을 상세하게 분석하여, 주의사항과 다음 진행 사항을 포함한 체계적인 git 커밋을 생성합니다.

## 실행 절차

### 1단계: 현재 상태 분석

다음 정보를 수집합니다:

1. **git status** - 변경된 파일 목록 (untracked 포함, `-uall` 플래그 사용 금지)
2. **git diff** - staged + unstaged 변경 내용
3. **git log** - 최근 커밋 메시지 (스타일 참고용)
4. **계획서 확인** - `.claude/plans/` 디렉토리의 관련 계획서를 읽어서 현재 진행 상황 파악

### 2단계: 작업 내역 분석

수집한 정보를 기반으로 다음을 정리합니다:

- **완료된 작업**: 이번 세션에서 구현/변경한 내용을 모듈별로 상세 정리
- **주요 구현 사항**: 핵심 로직, 아키텍처 결정 사항
- **주의사항**: 알려진 제약, 잠재적 이슈, 반드시 확인해야 할 사항
- **다음 진행 사항**: 아직 미구현이거나 추가 작업이 필요한 항목

### 3단계: 커밋 메시지 작성

아래 형식으로 커밋 메시지를 작성합니다:

```
{한줄 요약} (예: feat: 터틀 트레이딩 봇 핵심 모듈 구현)

## 완료된 작업
- [모듈명] 구현 내용 상세 설명
- [모듈명] 구현 내용 상세 설명
  ...

## 주요 구현 사항
- 핵심 로직이나 아키텍처 결정 사항
  ...

## 주의사항
- 알려진 제약이나 주의할 점
  ...

## 다음 진행 사항
- [ ] 아직 해야 할 작업 1
- [ ] 아직 해야 할 작업 2
  ...

Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>
```

### 4단계: 커밋 실행

1. 관련 파일을 staging에 추가합니다 (민감 파일 `.env` 등 제외)
2. 위 메시지로 커밋을 생성합니다
3. `git status`로 커밋 결과를 확인합니다

---

## 주의사항

1. **민감 파일 제외**: `.env`, 자격증명 파일 등은 절대 커밋하지 않습니다. 발견 시 사용자에게 경고합니다.
2. **계획서 참조**: `.claude/plans/` 의 계획서를 참조하여 "다음 진행 사항"을 계획서 기준으로 정리합니다.
3. **구체적 서술**: "기능 구현" 같은 모호한 표현 대신 "돈치안 채널 20일 돌파 매수 로직 구현" 처럼 구체적으로 작성합니다.
4. **한국어 사용**: 커밋 메시지는 한국어로 작성합니다.
5. **HEREDOC 사용**: 커밋 메시지는 반드시 HEREDOC 형식으로 전달하여 포맷이 깨지지 않도록 합니다.
6. **amend 금지**: 기존 커밋을 수정하지 않고 항상 새 커밋을 생성합니다.
