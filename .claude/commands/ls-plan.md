# ls-plan: 프로젝트 계획서 작성

사용자의 요청을 받아 체계적인 프로젝트 계획서를 작성합니다.

## 실행 절차

### 1단계: 사용자 입력 확인

사용자에게 다음을 질문합니다:
- **프로젝트명**: 계획서에 사용할 프로젝트 이름
- **목표**: 이 프로젝트가 달성하려는 것이 무엇인지
- **추가 컨텍스트**: 참고할 기존 코드, 문서, 요구사항 등

### 2단계: 계획서 파일 생성

`.claude/plans/` 디렉토리에 마크다운 파일을 생성합니다.

- **파일명 규칙**: `YYYY-mm-dd-{프로젝트명-slug}.md` (오늘 날짜 기준)
- 예시: `2026-03-25-user-auth-system.md`

### 3단계: 계획서 작성

아래 6개 Phase를 모두 포함하여 계획서를 작성합니다.
각 Phase는 **비개발자 투자자도 쉽게 이해할 수 있는 언어**로 작성합니다.
기술 용어가 필요한 경우 반드시 쉬운 설명을 병기합니다.

---

## 계획서 템플릿

```markdown

> 작성일: YYYY-mm-dd
> 작성자: Claude AI & {사용자}
> 브랜치: plan/{프로젝트명-slug}
> 상태: 🟡 계획 수립 중

---

## Phase 1: 의도 파악 (Intent Analysis)

### 무엇을 하려는 건가요?
- 프로젝트의 배경과 목적을 명확히 정리합니다.

### 핵심 질문
- 이 프로젝트가 해결하려는 문제는 무엇인가?
- 최종 사용자는 누구인가?
- 성공의 기준은 무엇인가?

### 결론
> (분석 결과를 여기에 작성)

### ✅ Phase 1 검토 요청
> 위 내용을 확인해 주세요. 의도와 목적이 맞는지 피드백을 주시면 다음 단계로 진행합니다.
```

### 4단계: Git 브랜치 생성

계획서 작성 후 다음 명령으로 작업 브랜치를 생성합니다:

```
git checkout -b plan/{프로젝트명-slug}
```

### 5단계: 진행 방식 안내

사용자에게 다음과 같이 안내합니다:

> 계획서가 `.claude/plans/YYYY-mm-dd-{프로젝트명}.md`에 저장되었습니다.
> 작업 브랜치 `plan/{프로젝트명}`이 생성되었습니다.
>
> 각 Phase마다 검토 요청(✅)이 있습니다.
> Phase별로 내용을 확인하시고 피드백을 주시면 다음 단계로 진행합니다.
> 모든 Phase가 완료되면 실제 구현을 시작합니다.

---

## 주의사항

1. **쉬운 언어 사용**: 모든 설명은 비개발자 투자자도 이해할 수 있어야 합니다.
2. **단계별 검토**: 각 Phase 완료 후 반드시 사용자 확인을 받고 다음으로 넘어갑니다.
3. **날짜 필수**: 파일명에 반드시 `YYYY-mm-dd` 형식의 날짜를 포함합니다.
4. **브랜치 필수**: 계획서 작성과 함께 반드시 Git 브랜치를 생성합니다.
5. **계획서 우선 저장**: 구현 전에 반드시 `.claude/plans/`에 계획서를 먼저 저장합니다.
