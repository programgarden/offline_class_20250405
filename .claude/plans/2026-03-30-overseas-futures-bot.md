## Phase 1: 의도 파악 (Intent Analysis)

### 무엇을 하려는 건가요?

기존에 만든 **해외주식 터틀 트레이딩 봇**에 **해외선물 모의투자 자동매매** 기능을 추가합니다.
같은 터틀 트레이딩 전략(추세 추종)을 사용하되, 대상을 주식이 아닌 **선물 계약**(나스닥 선물, S&P500 선물, 원유, 금 등)으로 확장합니다.

### 배경

- **해외선물이란?**: 미래 특정 시점에 정해진 가격으로 사고팔기로 약속하는 계약. 주식과 달리 **레버리지(적은 돈으로 큰 거래)** 가 핵심
- **왜 모의투자?**: 선물은 레버리지가 크므로 실전 전에 반드시 모의투자로 전략 검증 필요
- **왜 터틀 전략?**: 터틀 트레이딩은 원래 선물 시장에서 탄생한 전략. 주식보다 선물에 더 적합
- **왜 지금?**: 해외주식 봇이 안정화되어 같은 아키텍처를 재활용할 수 있음

### 핵심 질문과 답변

| 질문 | 답변 |
|------|------|
| 해결하려는 문제? | 터틀 전략의 원래 적용 대상인 선물 시장으로 확장 + 모의투자로 선물 전략 검증 |
| 최종 사용자? | ProgramGarden (개인 투자자) |
| 성공 기준? | 모의투자에서 터틀 전략이 선물 시장에서도 유효한지 데이터 확보. 안정적 24시간 운영 |
| 기존 시스템과의 관계? | 동일 프로젝트 내에서 해외주식과 **병렬 운영** (주식=실전, 선물=모의) |

### 프로젝트 범위

| 항목 | 포함 | 제외 |
|------|------|------|
| 선물 종류 | 주요 지수/원자재 선물 (NQ, ES, CL, GC 등) | 옵션, 스프레드 매매 |
| 매매 방식 | 터틀 전략 (돈치안 돌파 + 트레일링 스탑) | 스캘핑, 아비트리지 등 다른 전략 |
| 계좌 모드 | **모의투자만** | 실전 계좌 (추후 별도 계획) |
| 주문 유형 | 시장가 + 지정가 | MIT, 스탑 주문 |

### 해외주식과 해외선물의 핵심 차이점

| 항목 | 해외주식 | 해외선물 |
|------|----------|----------|
| 단위 | 주 (shares) | 계약 (contracts) |
| 레버리지 | 없음 (1:1) | 높음 (증거금 기반) |
| 만기 | 없음 | 있음 (분기별 롤오버 필요) |
| 거래 시간 | 정규장 + 주간거래 (~14시간) | 거의 24시간 (하루 1시간 휴장) |
| 손익 계산 | 주가 × 수량 | 틱 사이즈 × 틱 밸류 × 수량 |
| 리스크 | 투자금 이하 손실 | 투자금 초과 손실 가능 (레버리지) |
| 종목 선정 | 수천 개 중 스크리닝 | 사전 지정 (5~10개 주요 상품) |

### 어떤 정보를 모았나요?

#### 핵심 도구: programgarden-finance 해외선물 API

해외주식과 동일한 라이브러리에 해외선물 모듈이 이미 있습니다. 패턴도 거의 동일합니다.

| 필요 기능 | 라이브러리 지원 | 사용할 TR코드 / 기능 |
|-----------|----------------|---------------------|
| 로그인 (모의투자) | O | `LS().login(paper_trading=True)` |
| 선물 마스터(종목목록) 조회 | O | `o3101` (해외선물마스터조회) |
| 현재가 조회 | O | `o3105` (해외선물_현재가조회) |
| 호가 조회 | O | `o3106` (해외선물_현재가호가조회) |
| 차트 데이터 (일/주/월) | O | `o3108` (해외선물_차트일주월조회) |
| 차트 데이터 (분봉) | O | `o3103` (해외선물_차트분봉조회) |
| 예수금/잔고 조회 | O | `CIDBQ03000` (예수금잔고현황조회) |
| 미결제 잔고(포지션) 조회 | O | `CIDBQ01500` (미결제잔고내역조회) |
| 주문 내역 조회 | O | `CIDBQ01800` (주문내역조회) |
| 신규 주문 | O | `CIDBT00100` (신규주문) |
| 주문 정정 | O | `CIDBT00900` (정정주문) |
| 주문 취소 | O | `CIDBT01000` (취소주문) |
| 실시간 체결 | O | `OVC` (해외선물체결) |
| 실시간 호가 | O | `OVH` (해외선물호가) |
| 주문 체결 알림 | O | `TC1/TC2/TC3` (주문접수/응답/체결) |
| 실시간 손익 추적 | O | `account_tracker` (계좌추적기) |

#### 주요 API 패턴 (해외주식과 비교)

```python
from programgarden_finance import LS

ls = LS()
# 해외주식: ls.login(appkey=..., appsecretkey=...)           → 실전
# 해외선물: ls.login(appkey=..., appsecretkey=..., paper_trading=True) → 모의

# 해외주식 도메인
stock = ls.overseas_stock()
stock.market()   # g3101, g3106, g3190 ...
stock.chart()    # g3204
stock.accno()    # COSOQ02701, COSOQ00201
stock.order()    # COSAT00301
stock.real()     # GSC, GSH

# 해외선물 도메인 (패턴 동일!)
futures = ls.overseas_futureoption()
futures.market()  # o3101, o3105, o3106 ...
futures.chart()   # o3103, o3108
futures.accno()   # CIDBQ03000, CIDBQ01500
futures.order()   # CIDBT00100, CIDBT00900, CIDBT01000
futures.real()    # OVC, OVH, TC1/TC2/TC3
```

#### 주요 거래 대상 선물 상품

| 상품 | 심볼 패턴 | 거래소 | 1틱 가치 | 증거금 (약) | 특징 |
|------|-----------|--------|----------|-------------|------|
| 나스닥100 E-mini | NQ + 월코드 + 연도 (예: NQM26) | CME | $5.00 | ~$18,000 | 기술주 지수, 변동성 높음 |

| 마이크로 나스닥 | MNQ + 월코드 + 연도 | CME | $0.50 | ~$1,800 | NQ의 1/10, 소액 매매 적합 |
| 마이크로 S&P | MES + 월코드 + 연도 | CME | $1.25 | ~$1,300 | ES의 1/10, 소액 매매 적합 |
| 원유 (WTI) | CL + 월코드 + 연도 | NYMEX | $10.00 | ~$7,000 | 원자재 대표 |
| 금 | GC + 월코드 + 연도 | COMEX | $10.00 | ~$10,000 | 안전자산 |

> 예: NQM26 = 나스닥100 선물 2026년 6월물

## Phase 4: 알고리즘 설계 (Algorithm Design)

### 핵심 로직은 어떻게 동작하나요?

### A. 종목 관리 (주식과 가장 큰 차이)

**주식 방식**: 수천 개 → 스크리닝 → 5개 선정
**선물 방식**: 사전 지정 목록 → 만기 심볼 자동 관리

```python
# config.py에 거래할 선물 상품 목록 지정
FUTURES_SYMBOLS = [
    {"base": "NQ", "name": "나스닥100 E-mini", "exchange": "CME"},
    {"base": "ES", "name": "S&P500 E-mini", "exchange": "CME"},
    {"base": "MNQ", "name": "마이크로 나스닥", "exchange": "CME"},
    {"base": "CL", "name": "원유 WTI", "exchange": "NYMEX"},
    {"base": "GC", "name": "금", "exchange": "COMEX"},
]
```

**만기 심볼 자동 생성**:
```
현재 날짜가 2026-03-30이면:
  → 다음 만기월: 2026년 6월 (M)
  → NQ의 실제 심볼: NQM26

만기 2주 전에 자동 롤오버:
  → 기존 NQM26 포지션 청산
  → 새로운 NQU26 (9월물)로 교체
```

### B. 매매 알고리즘 (터틀 전략 - 선물 버전)

주식과 동일한 돈치안 돌파 전략이지만, **계약 수 계산**이 다릅니다.

**매수 조건 (진입)**:
```
현재가 > 20일 최고가 (돈치안 채널 상단 돌파)
→ 주식과 동일
```

**포지션 크기 결정 (핵심 차이)**:
```
선물은 "몇 계약"을 살지 계산:

1계약 리스크 = ATR × 틱 밸류 / 틱 사이즈
  예: NQ의 ATR=200포인트, 틱밸류=$5, 틱사이즈=0.25
  → 1계약 리스크 = 200 × 5 / 0.25 = $4,000

계약 수 = (예수금 × 리스크비율%) ÷ 1계약 리스크
  예: 예수금 $100,000, 리스크비율 2%
  → 계약 수 = ($100,000 × 2%) ÷ $4,000 = 0.5 → 반올림 → 1계약
```

> **리스크비율**: 한 종목에 대해 감수할 수 있는 최대 손실 비율. 주식(투자금 %)과 다르게 선물은 증거금 대비 리스크로 계산

**매도 조건 (트레일링 스탑)**:
```
매수 후 최고가 기록을 계속 갱신
최고가에서 3×ATR 떨어지면 매도 (주식과 동일)

단, 손익 계산이 다름:
  손익 = (매도가 - 매수가) × 틱밸류 / 틱사이즈 × 계약수
  예: NQ 19000에 매수, 19200에 매도, 1계약
  → 손익 = (19200 - 19000) × $5 / 0.25 × 1 = $4,000
```

### C. 리스크 관리 (선물 버전)

선물은 레버리지 때문에 **더 보수적인** 리스크 관리가 필요합니다.

```
매 1분마다 계좌 전체 손익률 체크

만약 일일손실 >= 증거금의 3%:
  → 마이너스 종목 청산 + 텔레그램 경고
  → 신규 매수 일시 중단

만약 일일손실 >= 증거금의 5%:
  → 전 종목 청산 + 당일 매매 완전 중단
  → 텔레그램 비상 알림

추가: 증거금 사용률 감시
  → 증거금 사용률 80% 초과 시 신규 진입 차단
```

### D. 텔레그램 명령어 (선물 추가분)

기존 주식 명령어는 그대로 두고, 선물 전용 명령어를 추가합니다.

| 명령어 | 설명 |
|--------|------|
| `/fstatus` | 선물 현재 상태 (포지션, 증거금, 손익) |
| `/fset` | 선물 파라미터 변경 (channel, atr, risk 등) |
| `/fsettings` | 선물 설정값 전체 보기 |
| `/fstop` | 선물 매매 일시 중단 |
| `/fstart` | 선물 매매 재개 |
| `/freport` | 선물 매매 리포트 |
| `/flist` | 거래 대상 선물 상품 목록 |

### E. 일일 스케줄 (선물)

| 한국시간 (서머타임) | 이벤트 | 동작 |
|---------------------|--------|------|
| 06:00 | 일일 마감 | 일일 리포트 정리 + 텔레그램 전송 |
| 06:00~07:00 | **휴장** | 정산 시간, 매매 없음 |
| 07:00 | 개장 | 로그인 재확인, 실시간 연결, 리스크 초기화 |
| 07:00~06:00 | 장중 (23시간) | 돌파 신호 감시 + 매매 실행 + 트레일링 스탑 |
| 매 3시간 | 헬스 체크 | API 세션 확인, 증거금 확인 |

> 주식(~14시간 거래)과 달리 선물은 **23시간 연속** → 스케줄이 훨씬 단순

### DB 테이블 설계 (선물용 추가)

```sql
-- 선물 포지션 (미결제 잔고)
CREATE TABLE futures_positions (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL UNIQUE,         -- 예: NQM26
    base_symbol TEXT NOT NULL,           -- 예: NQ
    direction TEXT NOT NULL,             -- LONG / SHORT
    quantity INTEGER NOT NULL,           -- 계약 수
    avg_entry_price REAL NOT NULL,       -- 평균 진입가
    highest_price REAL NOT NULL,         -- 매수 후 최고가 (트레일링용)
    lowest_price REAL NOT NULL,          -- 매수 후 최저가 (숏 트레일링용)
    trailing_stop_price REAL NOT NULL,   -- 현재 스탑 가격
    atr REAL NOT NULL,                   -- ATR 값
    tick_size REAL NOT NULL,             -- 틱 사이즈
    tick_value REAL NOT NULL,            -- 틱 밸류
    entry_date TEXT NOT NULL,
    updated_at TEXT DEFAULT (datetime('now'))
);

-- 선물 매매 기록
CREATE TABLE futures_trades (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL,
    base_symbol TEXT NOT NULL,
    direction TEXT NOT NULL,             -- LONG / SHORT
    order_type TEXT NOT NULL,            -- ENTRY / EXIT
    order_no TEXT,                       -- LS증권 주문번호
    quantity INTEGER NOT NULL,           -- 계약 수
    price REAL NOT NULL,
    pnl REAL,                           -- 실현 손익 (청산 시)
    reason TEXT,                         -- ENTRY / TRAILING_STOP / RISK_STOP / ROLLOVER
    executed_at TEXT DEFAULT (datetime('now'))
);

-- 선물 일일 리포트
CREATE TABLE futures_daily_reports (
    id INTEGER PRIMARY KEY,
    report_date TEXT NOT NULL UNIQUE,
    starting_equity REAL,               -- 장 시작 예탁자산
    ending_equity REAL,                 -- 장 종료 예탁자산
    daily_pnl REAL,                     -- 일일 손익
    daily_pnl_rate REAL,                -- 일일 수익률
    margin_used REAL,                   -- 사용 증거금
    margin_rate REAL,                   -- 증거금 사용률
    total_trades INTEGER,
    winning_trades INTEGER,
    losing_trades INTEGER,
    risk_stop_triggered INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);

-- 선물 상품 정보 (틱사이즈, 틱밸류 캐시)
CREATE TABLE futures_specs (
    base_symbol TEXT PRIMARY KEY,        -- NQ, ES, CL 등
    name TEXT NOT NULL,                  -- 상품명
    exchange TEXT NOT NULL,              -- CME, NYMEX, COMEX
    tick_size REAL NOT NULL,
    tick_value REAL NOT NULL,
    margin_required REAL,                -- 필요 증거금 (참고용)
    updated_at TEXT DEFAULT (datetime('now'))
);
```
