## Phase 1: 의도 파악 (Intent Analysis)

### 무엇을 하려는 건가요?

터틀 트레이딩 전략을 기반으로 **해외주식(나스닥/뉴욕거래소)을 자동으로 사고파는 프로그램**을 만듭니다.
LS증권 **실전 계좌**에서 24시간 돌아가며, 종목 분석부터 매매까지 전부 자동으로 처리합니다.
중요한 일이 생기면 텔레그램으로 알려줍니다.

### 배경

- **터틀 트레이딩**: 1980년대 전설적인 트레이더 리처드 데니스가 만든 전략. "추세를 따라가라"가 핵심. 가격이 일정 기간 최고점을 돌파하면 사고, 떨어지면 파는 방식
- **왜 실전?**: 해외주식은 실전 계좌로 바로 운영. 리스크 관리(4%/5% 일일 손실 한도)로 손실 제한
- **왜 자동화?**: 미국 시장은 한국 시간으로 밤~새벽에 열리고, 주간거래까지 포함하면 거의 하루 종일 열림. 사람이 직접 매매하기 어려움

### 핵심 질문과 답변

| 질문 | 답변 |
|------|------|
| 해결하려는 문제? | 미국 시장 시간대에 수동 매매 불가 + 감정 배제한 체계적 매매 필요 |
| 최종 사용자? | ProgramGarden (개인 투자자) |
| 성공 기준? | 드라이런에서 파라미터 튜닝 후 수익 구조 확인 → 실전 전환. 리스크 관리(4%/5%)는 항상 정상 동작 |
| 운영 방식? | 텔레그램 명령어로 모드 전환, 파라미터 변경, 상태 확인 등 원격 제어 |

### 프로젝트 범위

| 단계 | 내용 | 우선순위 |
|------|------|----------|
| 1단계 (이번) | 해외주식 **실전** 자동매매 | **지금** |
| 2단계 (추후) | 해외선물 모의투자 자동매매 | 나중에 |

## Phase 2: 자료 수집 (Research & Data Collection)

### 어떤 정보를 모았나요?

#### 핵심 도구: programgarden-finance 라이브러리

LS증권 API를 파이썬으로 쉽게 쓸 수 있게 만든 라이브러리입니다. 우리가 필요한 기능이 전부 있습니다.

| 필요 기능 | 라이브러리 지원 | 사용할 TR코드 / 기능 |
|-----------|----------------|---------------------|
| 로그인 & 인증 | O | `LS().login()` (실전) / `LS().login(paper_trading=True)` (선물 모의) |
| 현재가 조회 | O | `g3101` (현재가), `g3106` (호가) |
| 차트 데이터 (일/주/월) | O | `g3204` (차트일주월년별조회) |
| 시간대별 조회 | O | `g3102` (시간대별조회) |
| 계좌 잔고 | O | `COSOQ02701` (예수금), `COSOQ00201` (보유종목) |
| 주문 (매수/매도) | O | `COSAT00301` (신규주문) |
| 주문 정정/취소 | O | `COSAT00311` (정정) |
| 실시간 체결 | O | `GSC` (WebSocket 실시간 체결) |
| 실시간 호가 | O | `GSH` (WebSocket 실시간 호가) - 단, 잔량 데이터 제한 있음 |
| 계좌 실시간 모니터링 | O | `account_tracker` (실시간 손익 추적) |

#### 주요 API 패턴

```python
# 기본 구조
from programgarden_finance import LS
ls = LS()
ls.login(appkey="...", appsecretkey="...")  # 실전 (해외주식)
# ls.login(appkey="...", appsecretkey="...", paper_trading=True)  # 모의투자 (해외선물)

# 해외주식 도메인
stock = ls.overseas_stock()
stock.market()   # 시세 조회
stock.chart()    # 차트 데이터
stock.accno()    # 계좌 조회
stock.order()    # 주문
stock.real()     # 실시간 데이터

# 비동기 요청 패턴
result = await request.req_async()           # 단건 요청
await request.retry_req_async(callback)      # 재시도 포함
await request.occurs_req_async(callback)     # 페이징 자동 처리
```

#### 거래소 코드

| 코드 | 거래소 |
|------|--------|
| `81` | NYSE / AMEX |
| `82` | NASDAQ |

#### 종목 분석을 위한 외부 데이터

| 데이터 | 소스 | 용도 |
|--------|------|------|
| 나스닥/NYSE 종목 리스트 | LS증권 API 또는 외부 API | 분석 대상 종목 풀 |
| 일봉/주봉 차트 데이터 | LS증권 `g3204` | 추세 분석 (이동평균, 돈치안 채널) |
| 재무 데이터 | 외부 API (yfinance 등) | 재무 건전성 필터링 |

#### 제약 사항

| 항목 | 내용 |
|------|------|
| API 속도 제한 | 초당 3건 (SetupOptions으로 자동 대기 가능) |
| 실전 계좌 사용 | 해외주식은 실전 → 리스크 관리 로직이 반드시 정상 동작해야 함 |
| GSH 호가 제한 | 실시간 호가의 개별 잔량이 항상 0으로 옴 → REST `g3106` 사용 필요 |
| 미국 시장 시간 | 정규장: 한국시간 23:30~06:00 (서머타임 22:30~05:00) |

### 데이터 흐름

1. **분석 단계** (미국장 마감 후, 한국시간 07:00경)
   - 종목 분석기가 나스닥+NYSE 종목을 스크리닝
   - 추세 좋고 재무 건전한 상위 5개 종목 선정
   - 결과를 SQLite에 저장 + 텔레그램으로 분석 결과 전송

2. **주간거래 세션** (한국시간 09:00~16:30 / 비서머타임 10:00~18:00)
   - 계좌 예수금 확인 → 사용 가능 금액(50%) 계산
   - 분석된 종목 로드 + 종목당 투자금액 배분 계산
   - 돈치안 채널 돌파 시 **지정가 매수** 주문 실행 (주간거래는 지정가만 가능)
   - 주간거래 대상 약 1,000개 주요 종목만 거래 가능
   - ※ 미체결 주문은 세션 종료 시 자동 취소됨

3. **정규장 세션** (한국시간 23:30~06:00 / 비서머타임 22:30~05:00)
   - 프리마켓(18:00~)부터 모니터링 시작
   - 주간거래에서 미체결된 주문 재실행
   - 정규장에서 추가 매수/매도 실행 (시장가 주문 가능)
   - 매수 체결 즉시 트레일링 스탑 설정
   - 실시간 가격 모니터링

4. **리스크 감시** (모든 세션 공통)
   - 실시간으로 계좌 전체 손익률 추적
   - 4% 손실 도달 → 마이너스 종목 청산 + 텔레그램 경고
   - 5% 손실 도달 → 전종목 청산 + 당일 매매 중단

5. **정리** (미국장 마감 후, 한국시간 06:00~07:00)
   - 당일 매매 결과 정리 → SQLite 저장
   - 텔레그램으로 일일 리포트 전송

> **참고**: 해외주식은 실전 계좌이므로 주간거래(ATS) 포함 모든 세션 이용 가능.
> 주간거래는 지정가 주문만 가능하며, 약 1,000개 주요 종목만 거래 가능한 점 유의.

## Phase 4: 알고리즘 설계 (Algorithm Design)

### 핵심 로직은 어떻게 동작하나요?

### A. 종목 선정 알고리즘

**비유**: 수천 명의 지원자 중에서 5명을 뽑는 면접 과정. 1차 서류심사(재무) → 2차 실기(추세) → 최종 합격(5명)

**Step 1: 종목 풀 수집**
- 나스닥 + NYSE 상장 종목 리스트 확보
- 시가총액 하위 종목, 거래량 극소 종목 제외 (쓰레기 필터링)

**Step 2: 재무 건전성 필터 (1차 서류심사)**
- 부채비율 200% 이하 (빚이 너무 많지 않은 회사)
- 영업이익 흑자 (실제로 돈을 벌고 있는 회사)
- 시가총액 10억 달러 이상 (너무 작은 회사 제외)
- PER 0~50 사이 (적자기업이나 버블 수준 제외)

**Step 3: 추세 분석 (2차 실기심사)**
- 최근 3개월 수익률 계산 (모멘텀)
- 20일 이동평균 > 60일 이동평균 (상승추세 확인)
- ATR(평균진폭) 계산 (변동성 적절한지 확인)
- 돈치안 채널 20일 상단 근처에 있는지 (돌파 임박 종목 우선)

**Step 4: 최종 선정**
- 모멘텀 점수 + 재무 점수를 합산하여 상위 5개 선정
- SQLite에 저장 + 텔레그램 알림

### B. 매매 알고리즘 (터틀 전략 변형)

**비유**: "물이 둑을 넘으면 따라가고, 수위가 내려가면 빠져나오는" 전략

**매수 조건 (진입)**:
```
현재가격 > 20일 최고가 (돈치안 채널 상단 돌파)
→ "가격이 최근 20일 중 가장 높다 = 상승 추세 시작 신호"
```

**포지션 크기 결정**:
```
1종목당 투자금 = (예수금 × 50%) ÷ 보유 예정 종목수
최대 5종목, 예수금 부족 시 종목 수 축소
```

예시: 예수금 $10,000 → 사용가능 $5,000 → 종목당 $1,000

**매도 조건 (트레일링 스탑)**:
```
매수 후 최고가 기록을 계속 갱신
최고가에서 3×ATR(평균진폭)만큼 떨어지면 매도
→ "이익은 최대한 달리게 하고, 떨어지기 시작하면 빠져나오는" 방식
```

예시:
- $100에 매수, ATR이 $2이면 스탑가격 = $100 - $6 = $94
- 가격이 $110까지 오르면 스탑가격 = $110 - $6 = $104
- 가격이 $104 아래로 떨어지면 매도 → 수익 $4 확보

### C. 리스크 관리 알고리즘

**비유**: 자동차의 브레이크 시스템. 일반 브레이크(4%)와 비상 브레이크(5%)

```
매 1분마다 계좌 전체 손익률 체크

만약 일일손실률 >= 4%:
   → 당일 마이너스인 종목들만 전량 매도
   → 텔레그램: "⚠️ 4% 손실 경고: OO, OO 종목 청산"
   → 신규 매수 일시 중단

만약 일일손실률 >= 5%:
   → 전 종목 전량 매도
   → 당일 매매 완전 중단
   → 텔레그램: "🚨 5% 비상 청산: 전종목 매도, 당일 매매 중단"
```

### D. 텔레그램 명령어

텔레그램을 **양방향 컨트롤러**로 사용합니다. 알림을 받을 뿐 아니라, 명령어로 시스템을 원격 제어할 수 있습니다.

| 명령어 | 설명 |
|--------|------|
| `/help` | 전체 명령어 목록 + 설명 |
| `/status` | 현재 상태 (보유종목, 손익, 모드) |
| `/mode` | 현재 모드 확인 (드라이런/실전) |
| `/mode live` | 실전 모드로 전환 |
| `/mode dry` | 드라이런 모드로 전환 |
| `/set channel 20` | 돈치안 채널 기간 변경 |
| `/set atr 3.0` | 트레일링 스탑 ATR 배수 변경 |
| `/set stocks 5` | 종목 수 변경 |
| `/set ratio 50` | 예수금 사용 비율(%) 변경 |
| `/settings` | 현재 설정값 전체 보기 |
| `/stop` | 매매 일시 중단 |
| `/start` | 매매 재개 |
| `/report` | 최근 매매 리포트 |

**운영 흐름**: 처음에 드라이런 모드로 시작 → 텔레그램으로 결과 확인 → 파라미터 튜닝(`/set`) → 수익 구조 확인 후 `/mode live`로 실전 전환

### DB 테이블 설계

```sql
-- 종목 분석 결과
CREATE TABLE stock_analysis (
    id INTEGER PRIMARY KEY,
    analysis_date TEXT NOT NULL,        -- 분석 일자
    symbol TEXT NOT NULL,               -- 종목 코드
    exchange_code TEXT NOT NULL,        -- 거래소 (81=NYSE, 82=NASDAQ)
    company_name TEXT,                  -- 회사명
    momentum_score REAL,               -- 모멘텀 점수
    financial_score REAL,              -- 재무 점수
    total_score REAL,                  -- 종합 점수
    selected INTEGER DEFAULT 0,        -- 선정 여부 (0/1)
    created_at TEXT DEFAULT (datetime('now'))
);

-- 매매 기록
CREATE TABLE trades (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL,
    exchange_code TEXT NOT NULL,
    order_type TEXT NOT NULL,           -- BUY / SELL
    order_no TEXT,                      -- LS증권 주문번호
    quantity INTEGER NOT NULL,
    price REAL NOT NULL,
    amount REAL NOT NULL,              -- 총 금액
    reason TEXT,                        -- 매매 사유 (ENTRY/TRAILING_STOP/RISK_STOP 등)
    executed_at TEXT DEFAULT (datetime('now'))
);

-- 현재 포지션 (보유 종목)
CREATE TABLE positions (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL UNIQUE,
    exchange_code TEXT NOT NULL,
    quantity INTEGER NOT NULL,
    avg_buy_price REAL NOT NULL,       -- 평균 매수가
    highest_price REAL NOT NULL,       -- 매수 후 최고가 (트레일링용)
    trailing_stop_price REAL NOT NULL, -- 현재 스탑 가격
    atr REAL NOT NULL,                 -- ATR 값
    entry_date TEXT NOT NULL,
    updated_at TEXT DEFAULT (datetime('now'))
);

-- 일일 리포트
CREATE TABLE daily_reports (
    id INTEGER PRIMARY KEY,
    report_date TEXT NOT NULL UNIQUE,
    starting_balance REAL,             -- 장 시작 잔고
    ending_balance REAL,               -- 장 종료 잔고
    daily_pnl REAL,                    -- 일일 손익
    daily_pnl_rate REAL,               -- 일일 수익률
    total_trades INTEGER,              -- 총 매매 횟수
    winning_trades INTEGER,            -- 수익 매매
    losing_trades INTEGER,             -- 손실 매매
    risk_stop_triggered INTEGER DEFAULT 0,  -- 리스크 청산 발동 여부
    created_at TEXT DEFAULT (datetime('now'))
);

-- 설정값
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    description TEXT,
    updated_at TEXT DEFAULT (datetime('now'))
);
```
